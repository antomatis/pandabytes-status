pandabytes — MCP quickstart

Get your pandabytes key — 3 steps

One read-only key in under a minute. The SAME key powers the MCP connector (paste one URL into claude.ai, Claude Code, Desktop or Cursor) and the REST API (send it as X-API-Key / ?key=). Self-serve for @boredpanda.com addresses — your inbox is the gate, no operator approval.

1

Enter your work email

We email a one-click confirm link to your @boredpanda.com address.

Only @boredpanda.com emails are accepted here.

2

Check your email and click Confirm

We just sent a confirmation email. Open it and click the big Confirm button — that mints your key and shows your MCP URL.

Didn't get it? Check spam, or resend the link.
3

Paste your URL into Claude

Your ready-to-paste MCP URL (it carries your read-only key — keep it private):

https://pandabytes.restful.zone/mcp?k=<your key>

This same key is your REST API key. The part after k= is your key — send it as the X-API-Key header (or ?key=) to call the REST API directly.

Open claude.ai connectors →
  1. On claude.ai/customize/connectors, choose Add custom connector.
  2. Name it Pandabytes; paste the URL above as the Remote MCP server URL; leave OAuth Client ID / Secret blank.
  3. Save, then ask Claude "which pandabytes sources are stale?"
Other clients (Claude Code, Desktop, Cursor) & what the connector can do

The same URL works everywhere — Streamable HTTP, nothing to install. Once you have your URL from step 3:

Claude Code (CLI):

claude mcp add --transport http pandabytes \
  "https://pandabytes.restful.zone/mcp?k=<your key>"

Claude Desktop / Cursor: add it as a remote / HTTP MCP connector pointed at the same URL.

The connector is read-only with exactly three tools — describe_schema() (the data map, call it first), query(sql) (one sandboxed read-only SELECT over the whole catalog), and agentic_query(question) (multi-step explorer). It never triggers an upstream fetch, so any query volume is safe. Full tool reference + schema is in the API docs.

Just want REST, not MCP? You already have it — the key inside your step 3 URL (after k=) is your REST API key. Send it as X-API-Key or ?key=; see the API docs.
