What is pandabytes?

pandabytes is BoredPanda's central content-performance data hub — a single source of truth that unifies editorial publishing, audience analytics, social reach, ad data, and the Reddit reference corpus into one DuckDB + parquet catalog. Every upstream source (GA4, Chartbeat, MSN, AOL, Google News, Pinterest, Facebook, the gateway publishing spine, Reddit, and more) is ingested into tidy daily facts, with a freshness view that tells you, at a glance, which sources are current and which need attention.

The hub exposes a small, read-only HTTP/JSON REST API designed for AI agents and dashboards. An API call never triggers an upstream fetch — it only reads the already-built catalog — so any query volume is safe and quota-free. This site is a static, keyless front-end over that API: the dashboard renders live from a same-origin /snapshot.json and needs no key. The key-gated REST API below is for agents, MCP clients, and programmatic dashboards.

Status & kind enums

Every source has a kind and a freshness status:

statusmeaning
FRESHdata is within its cadence window
STALEnewest complete day is older than the stale threshold
STALLEDwas fresh, but no new rows for several cadences (not yet past stale)
MISSINGactive in the catalog but 0 rows on the box
FROZENintentionally not updated (historical / locked source)
DEPRECATEDcatalog lifecycle is deprecated / retired

kind is one of: spine · metric · reference · vector · federated · derived.

How to get an API key

The API uses a single read-only key, passed as the X-API-Key header (or the ?key= query param — the query param wins if both are present). GET /v1 and GET /v1/health are public so an agent can bootstrap before it has a key.

  1. Request a key on the Get a key page, or POST your email directly:
curl -X POST "https://pandabytes.restful.zone/v1/keys/request" \
     -H "Content-Type: application/json" \
     -d '{"email":"you@company.com"}'
{ "ok": true, "data": { "status": "pending", "email": "you@company.com",
  "request_id": "req_7Fa1c9d2",
  "message": "approval link emailed to the operator; you'll receive your key once approved" } }
  1. The operator approves by email; your key is emailed to you (never returned in a body).
  2. Pass it as the X-API-Key header (or ?key=) on the requests below, or in your MCP client config. The dashboard itself stays keyless.

REST API usage

Base URL: https://pandabytes.restful.zone/v1. Every response uses the same envelope: { "ok": true, "data": …, "meta": { "generated_at": … } }, and every error is { "ok": false, "error": …, "code": …, "hint": … }.

Bootstrap (public, no key)

curl https://pandabytes.restful.zone/v1
curl https://pandabytes.restful.zone/v1/health
{ "ok": true, "data": {
    "status": "ok", "db_ok": true, "db_path": "/DATA/duckdb/pandabytes.db",
    "catalog_built_at": "2026-06-16T13:01:00+03:00", "catalog_age_minutes": 11,
    "sources_total": 22 },
  "meta": { "generated_at": "2026-06-16T13:12:00+03:00" } }

30-day coverage — GET /v1/sources/{id}/coverage?days=30

One row per calendar day (default 30, max 365). Empty days come back as rows: 0, key_metric: null so the x-axis is gap-free — this is exactly what the dashboard's column charts plot to surface missing-data days.

Editorial performance

The editorial KPI layer reports content performance over a rolling window (default days=30, max 365) across the unified analytics facts (GA4 + Chartbeat). Use it for Chartbeat-style reporting: top authors, top categories, top articles, and a single article's full performance profile. All four are key-gated reads — they never touch upstream.

Top articles — GET /v1/articles/top

Ranks articles by a metric over the window. Params: ?metric= (pageviews default · engagement_time · sessions · timespent), ?days= (default 30), ?limit= (default 50, max 200).

curl -H "X-API-Key: YOUR_KEY" "https://pandabytes.restful.zone/v1/articles/top?metric=pageviews&days=30&limit=2"

By team — GET /v1/perf/team

Grouped by editorial team_id. Currently a stub: the author/category → team mapping tables are not yet populated, so every article resolves to team_id=0 (unassigned). Error envelope

{ "ok": false, "error": "missing or invalid API key", "code": "unauthorized",
  "hint": "pass ?key=YOUR_KEY or the X-API-Key header; request one via POST /v1/keys/request" }

Stable code values: unauthorized, source_not_found, bad_request, invalid_param, db_unavailable, token_spent, rate_limited, internal.

pandabytes MCP server

The hub ships an MCP (Model Context Protocol) server so AI agents can query the catalog natively. Install it with uvx and point it at your key:

uvx mcp-pandabytes

Configure it in your MCP client (e.g. Claude Desktop / Claude Code mcp.json):

{
  "mcpServers": {
    "pandabytes": {
      "command": "uvx",
      "args": ["mcp-pandabytes"],
      "env": {
        "PANDABYTES_API_KEY": "YOUR_KEY",
        "PANDABYTES_API_BASE": "https://pandabytes.restful.zone/v1"
      }
    }
  }
}

The MCP exposes the same read-only surface as the REST API — both the freshness tools (source board, source detail, 30-day coverage, hub stats) and the editorial KPI tools (top articles, per-article performance, top authors / categories / teams) — so an agent can ask "which pandabytes sources are stale?", "chart GA4 pageviews for the last 30 days", or "who were our top authors by pageviews this month?" without writing any HTTP code.

Agent quick-start (3 calls)

# 1) discover the surface (no key)
curl https://pandabytes.restful.zone/v1
# 2) request a key, get it approved by email, then:
KEY=YOUR_KEY
# 3) read the board, drill in, chart it
curl -H "X-API-Key: $KEY" "https://pandabytes.restful.zone/v1/sources"
curl -H "X-API-Key: $KEY" "https://pandabytes.restful.zone/v1/sources/ga4"
curl -H "X-API-Key: $KEY" "https://pandabytes.restful.zone/v1/sources/ga4/coverage?days=30"
curl -H "X-API-Key: $KEY" "https://pandabytes.restful.zone/v1/stats"
# 4) editorial reporting (top articles / authors / categories)
curl -H "X-API-Key: $KEY" "https://pandabytes.restful.zone/v1/articles/top?metric=pageviews&days=30"
curl -H "X-API-Key: $KEY" "https://pandabytes.restful.zone/v1/perf/author?days=30&limit=10"
